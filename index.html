Git & GitHub
Why are we learning this?
  • +Open collaboration, open source, open communities
  • +To build good habits early
  • +In the long term it will save you time and energy
Git
What is Git?
  • +Git is a Version Control System (VCS) that tracks file changes in a directory
  • +You can think of git as a set of commands that help you track changes to your code
  • +The way we use these commands is via a GUI (graphical user interface), like GitHub Desktop
  • +Or with command line, using the CLI (command line interface)
GitHub
What is GitHub?
  • +GitHub is a hosting service for your code repositories
  • +GitHub comes with some features, like GitHub pages, which we'll use in an upcoming exercise
  • +There are other hosting services out there (Bitbucket, GitLab)
Git Basics I: Cloning your first repo
Basic terminology & commands
  • +We will start by 'cloning' a remote repo (today's code examples)
  • +Remote repo: a repository that is hosted somewhere (like GitHub)
  • +Cloning a repo: making a copy of it locally (downloaded to your machine)

Running a local server
Why do we need to do this?
  • +You should get in the habit of using a local server, even when working with static files
  • +You won't be able to access your file system (e.g, if you need to load in an image)
  • +You won't be able to make http requests, like when working with an API
  • +READING (to puruse): MDN Web Docs - Setup a Local Testing Server
Running a local server
From your Editor
  • +Editors will either have this functionality built in or have a package that you can install
  • +Packages: utilities you can install for your editor to aid in development (share your faves!)
  • +Atom Live Server package, VS Code Live Server package

What is the DOM?
Document Object Model
  • +A programming interface for HTML and XML documents
  • +Represents a webpage so that programs can change the document structure, style, and content
  • +The DOM represents the document as nodes and objects
  • +REFERENCE: CSS Tricks: What is the DOM?
  • +REFERENCE: MDN Web Docs: Intro to the DOM
